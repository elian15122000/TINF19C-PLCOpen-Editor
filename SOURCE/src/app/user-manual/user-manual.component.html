User Manual


1. Navigator

1.1. Usage





There is a bar at the top of each (sub) page as you can see in the image above. On it there is a house icon for navigation back to the homepage. Next to the home button is an about button, which you can use to get to the about page, unless you are in the editor. In addition, you can always see the name of the project or product including version (currently PLCOpen-Editor 1.0) on the bar.



On the homepage you can click on three boxes that are shown in the picture above. By clicking on them you can create a new project, upload and open an existing project or access the user manual. If you decide to create a new project click the button 'New Project' and an input will open which you can see in the following picture.



Here you can enter the project name and then open it. If you decide to upload an existing project click the button 'Open Project'. Here you can upload a file to open it as you can see in the picture below.



If a project is opened (either by creating a new one or uploading an existing one), you can see the overview of the different POUs (programs) of the project.



Here you can add a new POU, delete an existing POU and also export your project by clicking the button 'export'. Furthermore you can change to another project by clicking the button 'Open another project'. If you decide to add a new POU, you will be asked to enter a name, the type and the language of the POU.



For a language you can choose between Sequential Function Chart (SFC) and Function Block Diagram (FBD). For a type only "program" is supported at the moment. To open a POU and therefore get to the editor you simply have to click on the name of the POU.

2. Converter

2.1. Import

Click on 'Open Project' on the homepage to upload an existing file as described in Chapter 1. Choose the file and click 'open'. By clicking this button the file gets imported. The POUs get read and appear in the POU list. Click then on a specific POU to load the POU and illustrate it on the editor.

2.2. Export

To export the xml file you will have to go back to the POU overview, where there is an export button. By clicking the button, the xml-file of the project gets exported.




3. Editor

3.1. Adding a function block





On the right side of the window there is a library where you can choose a function block that you want to add. By clicking 'add to the editor' it will appear on the left side in the editor. By using drag and drop you can move the position of the function block within the editor. An example of an added SR and RS block is shown in the picture below.



3.2. Adding a variable





Click the '+'-button to declare a new variable. There you can set a name, a class, an IEC-address, an initial value, options and you may add a documentation. Note that the field documentation is not supported yet and will therefore not be saved to the xml-file! All other attributes will be saved to the xml file. The name will be the name of the variable. For the field class you may choose where the variable shall be created (e.g. local or external). The field type will be the type of the variable like for example a bollean or an integer. Moreover you can set the IEC-address of the variable and you may set an initial value. You can also delete variables by clicking the delete icon. To add a variable to the editor, you can click the button 'add variable'.



Here you can choose the variable that you want to add, and you can choose the type of the variable (if it has an input, output or both) and if it shall be negated. By clicking the button 'add' the variable will appear on the editor.

3.3. Adding connections





Click on the element, you want to add a connection to. There will appear dropdown lists as shown in the image above, where you can choose what the input and output of a variable shall be connected with. Note that this feature is not implemented fully yet and bugs might appear.

3.4. Further actions

While editing you can press the button with the save icon.



This button will cache your changes you made in the editor. Furthermore there is a refresh icon next to the save button.



By clicking this button the chart will be updated.
